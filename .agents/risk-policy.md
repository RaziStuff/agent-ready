# Agent Risk Policy

Generated by agent-ready. Treat this as a starting point and edit it with repo-specific guidance.

## Risk Areas

- `.github/workflows` (medium): CI changes can affect validation and release behavior.
- `.github/workflows/ci.yml` (medium): CI changes can affect validation and release behavior.

## Default Agent Rules

- Do not read or expose secret-like files.
- Avoid hand-editing generated outputs.
- Treat migrations, infra, deployment, and CI changes as high-impact.
- Keep lockfile changes scoped to dependency changes.
- Run the smallest relevant validation before handing off.
